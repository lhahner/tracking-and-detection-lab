\subsection{Datasets}
The choice of dataset is important to correctly reflect the actual performance of different permutations of detection and tracking systems. Especially newer datasets that include night-time scenes, non-urban environments, or highly occluded images can provide useful insights into performance improvements over time. Furthermore, this section highlights the evolution of various dataset collections and challenges.

\subsubsection{MOT15 Dataset collection}
Different permutations of tracking and detection systems were evaluated on the MOT15 dataset \cite{MOTChallenge2015}. The MOT15 challenge is a dataset framework that contains 5503 training frames and 5783 testing frames, divided into a total of 22 sequences. The dataset includes both 3D and 2D sequences; however, for the current benchmark, only the 2D sequences are considered. 

Furthermore, the dataset provides a diverse set of scenes and categorizes the data according to camera movement (static or moving), viewpoint height, and weather conditions. The ground truth annotations are primarily generated through manual human labeling, supported by annotation tools such as the VATIC annotation tool. Some sequences originate from existing datasets, such as the ETH \cite{} dataset and the KITTI \cite{} dataset, which provide their own annotated data. In Figure \ref{fig:groundtruth}, exemplary ground truth annotations are presented.

This figure shows one of the targets used in the benchmark. Most sequences, such as KITTI-17, TUD-Crossing, and ETH-Crossing, depict urban environments. The dataset mainly consists of scenes in which multiple pedestrians are visible. As a result, occlusion is common due to overlapping pedestrians. Many sequences originate from datasets such as ETH-Bahnhof, ETH-Sunnyday, and ETH-Pedcross2.

As previously mentioned, the dataset is divided into two parts: the training set and the testing set. While the training and testing sets contain data from similar environments, they are recorded at different times. For example, the KITTI-17 sequence is from 2017 and is located in the training set, while the KITTI-19 sequence is from 2019 and is located in the testing set. This structure makes it possible to train and evaluate detection and tracking models under similar but temporally distinct conditions. Since this report focuses on pre-trained models, the evaluation is independent of whether the training or testing dataset is used.

\subsubsection{MOT20 Dataset collection}
Since the MOT15 dataset primarily includes daytime scenes with moderate crowd density, an additional dataset from a newer MOTChallenge benchmark is considered. The MOT20 dataset \cite{dendorfer2020mot20benchmarkmultiobject} includes scenes recorded during both daytime and nighttime, and in significantly more crowded environments. This provides additional challenges for detection and tracking systems and allows evaluation under more difficult and realistic real-world conditions.

The MOT20 dataset consists of a total of 8 sequences, split into 4 training and 4 testing sequences. The training set contains 8,931 frames, while the test set contains 4,479 frames. Compared to the MOT15 challenge, the pedestrian density per image has increased substantially, reaching peak densities of up to 246 pedestrians per frame. Furthermore, the dataset contains approximately three times more bounding box annotations than the MOT17 dataset, significantly increasing the complexity of the tracking task. This increase in annotation density and scene complexity is visible in the ground truth example shown in Figure~\ref{fig:groundtruth20}.

The high pedestrian density introduces additional challenges for MOT systems, as frequent occlusions between individuals occur. This makes it more difficult to maintain correct identity assignment and to associate object tracks consistently across multiple frames. As illustrated in Figure~\ref{fig:groundtruth20}, the dataset includes both indoor and outdoor scenes, recorded from elevated static camera viewpoints. These scenes contain a wide range of environmental conditions, including daytime and nighttime recordings, varying illumination conditions, and different crowd dynamics.

Additionally, all sequences are recorded in high resolution, typically up to 1920$\times$1080 pixels, allowing precise spatial localization of pedestrians. The dataset provides manually annotated ground truth bounding boxes, including unique object identities and visibility information. This enables a detailed evaluation of tracking performance, particularly in challenging scenarios with partial and full occlusions.

% Faster R-CNN
\subsubsection{Faster R-CNN}
Fast \ac{R-CNN} (2014) \cite{fast-r-cnn} and Faster R-CNN (2016) \cite{faster-r-cnn} build upon one of the earliest deep learning–based object detection systems, R-CNN \cite{r-cnn} (2013).  
The original R-CNN approach first generates region proposals from an input image using the selective search algorithm \cite{felzenszwalb}. Each proposed region is then passed through a CNN to extract features and classify the object within the region. Since a separate forward pass through the CNN is required for every region proposal, the method does not share computations between proposals and is therefore computationally expensive \cite{fast-r-cnn}.
Fast R-CNN addresses this limitation by processing the entire image with a CNN only once. Region proposals generated by selective search are then mapped onto the resulting feature map, where \ac{RoI} pooling extracts fixed-size feature representations for classification and bounding box regression.
Faster R-CNN further improves this architecture by replacing the selective search algorithm with a \ac{RPN}. The RPN generates candidate object regions directly from the feature maps and predicts both class probabilities and bounding box coordinates, which are subsequently refined by the Fast R-CNN detection network \cite{faster-r-cnn}.
All of these models are considered as convolutional deep-learning models visible in figure \ref{fig:timetable-detection}

% YOLO
\subsubsection{YOLOv1-12}
The YOLO detection system comes in various version, where each represents one step of improvements.
Compared to Faster R-CNN, YOLO does not generate region proposal first, instead it
tackles object tracking as a single regression problem, straight from image pixels 
to bounding box coordinates and class probabilities. Initally YOLO divides the input
image in an $N \times N$ grid. Each cell predicts bounding boxes, confidence scores 
and class probabilities. Making it a realtime single-stage detection system. \cite{yolo}
Further advancements of YOLO progressively addressed its limitations like for example the detection accuracy espacilly for small ones. 
One advancment was introduced by YOLOv2 a new 30-layer architecture with anchor boxes from Faster R-CNN and batch normalization to speed up convergence and enhance model performance. \cite{yolov2}
YOLOv3 then used different scales for detections for espacially small object to captures 
essential features at different resolutions. \cite{yolov3}
Later models such as YOLOv4 \cite{yolov4} and YOLOv5 \cite{yolov5} focused on architectural optimizations, improved feature aggregation, and more efficient training pipelines.
From YOLOv6 \cite{yolov6} onward, the focus shifted toward scalability and deployment efficiency, with techniques such as improved label assignment, self-distillation, and enhanced backbone structures. More recent versions, including YOLOv9 \cite{yolov9} and YOLOv10 \cite{yolov10}, introduced innovations such as programmable gradient information and the removal of non-maximum suppression to reduce latency. The latest models, YOLOv11 \cite{yolov11} and YOLOv12 \cite{yolov12}, further advance the architecture through attention mechanisms and optimized feature aggregation networks, achieving state-of-the-art detection accuracy while maintaining real-time inference speeds

The presented benchmark implements Yolov3, -v5, -v8, -v9, -v10, -v11, -v12 and -v26.
While Yolov26 is the latest implementation of YOLO from ultralytics. \cite{bibid} That introduces
an edge-optimized model which introduces stable and faster inference using the MuSGD (SGD and Muon hybrid) optimizer \cite{yolov26}.

% DETR
\subsubsection{DETR}

Besides having a two-stage detection system with Faster R-CNN, we got a one-stage detection system
with YOLO now we take a closer look at transformer based detection systems as of DETR \cite{detr}.
The DETR networks uses a CNN as a backbone feature extractor, than passes the extracted features
to transformer encoder-decoder model. Each encoder layer has a standard architecture and consists of a multi-head self-attention module and a feed forward network (FFN). Essentially the 
encoder produces object queries that represent positional encoding from the input features.
These are then independtly decoded into box coordinates and class lables by the feed forward
network (FFN). The FFN predicts the normalized center
coordinates, height and width of the box w.r.t. the input image, and the lin-
ear layer predicts the class label using a softmax function. DETRs architecture benefits 
from it's simplitfy and can be extended to other tasks as of object segmentation.

\subsection{Multi-object-tracking systems}

The benchmark measures the performance of various permutations of detection and tracking systems, the following presents SORT and DeepSORT which are both implemented in the given benchmark.

\subsubsection{SORT}
\ac{SORT} is a tracking system introduced in 2017 that, as the name suggests, focuses on online and real-time tracking. In an online setting, the tracker processes frames sequentially and only has access to the current and previous frames while tracking detected objects in real time. The objects are typically detected by an underlying detection system, such as one of the detectors presented previously. SORT predicts the next possible position of each object in the subsequent frame and associates the new detections with existing tracks based on their bounding boxes. The prediction of the next object position is performed using a Kalman filter framework. Subsequently, SORT computes the \ac{IOU} distance between each detection from the chosen detector and the predicted bounding boxes generated by the Kalman filter. This matching step is referred to as the assignment problem.
The assignment between detections and predicted tracks is solved optimally using the Hungarian algorithm. A track is terminated if the corresponding object is not detected for a predefined number of consecutive frames. SORT is implemented for the benchmark and combined with 
all the named detection systems.

\subsubsection{DeepSORT}
\ac{DeepSORT} builds upon the SORT framework for multi-object tracking. 
While SORT relies solely on motion information for track association, 
DeepSORT incorporates appearance-based matching using deep learning. 
Specifically, DeepSORT introduces a CNN-based feature extractor that 
generates an appearance embedding for each detected object.
These appearance features are used in addition to the motion model to 
improve the reliability of the data association step. In SORT, tracks 
may be incorrectly swapped when objects become occluded or when multiple 
objects with similar motion patterns are present. By incorporating visual 
appearance information, DeepSORT can better distinguish between objects 
with similar trajectories and therefore significantly reduces identity 
switches during tracking.
